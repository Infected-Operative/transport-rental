# Система аренды транспорта (велосипеды и самокаты)

## Описание проекта

**Предметная область:** Учёт парка городского транспорта (велосипеды и электросамокаты), управление доступностью, бронирование пользователями и администрирование системы.

**Реализованный функционал:**
- Регистрация и авторизация пользователей
- Хранение паролей в зашифрованном виде (хэши)
- Система ролей: обычный пользователь и администратор
- Полный CRUD-интерфейс для управления пользователями (администратор)
- Полный CRUD-интерфейс для управления транспортом (администратор)
- Фильтрация транспорта по статусу (доступен / арендован / на обслуживании) — доступна всем пользователям
- Статистика на главной странице: общее количество, доступно, арендованно, на обслуживании
- Заглушка бронирования для обычных пользователей (кнопка с уведомлением)
- Единый современный дизайн на всех страницах
- Защита маршрутов по ролям

**Технологии:**
- Backend: Flask (Python)
- База данных: SQLite (через Flask-SQLAlchemy)
- Аутентификация: Flask-Login
- Формы: Flask-WTF
- Шаблоны: Jinja2
- Стили: чистый CSS

**Архитектура:** Проект строго следует паттерну **MVC**:
- **Models** — `models.py` (User, Transport)
- **Views** — шаблоны в папке `templates/`
- **Controllers** — маршруты и логика в `app.py`

## Структура проекта
- transport_rental/
- ├── app.py                  # Основное приложение (контроллеры + маршруты)

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/Infected-Operative/transport_rental.git
cd transport_rental
```

### 2. Создание виртуального окружения (рекомендуется)
```Bash
python -m venv venv
```
## Активация:
### Windows:
venv\Scripts\activate
### Linux/Mac:
source venv/bin/activate

### 3. Установка зависимостей
```Bash
pip install -r requirements.txt
```

### 4. Запуск приложения
```Bash
python app.py
```
Приложение будет доступно по адресу: http://127.0.0.1:5000

## Запуск тестов:
```Bash
pytest tests/test_app.py -v
```
Ожидаемый результат: 25 passed
